CSS Priority and Overriding Properties

When learning CSS, it's important to understand how different properties interact with each other, and which ones take precedence or can override others. Hereâ€™s a list of important priority rules and overriding properties you should be aware of in CSS:

1. CSS Specificity

What it is: Specificity determines which CSS rule is applied when multiple rules target the same element.

Order of specificity (from lowest to highest):

  • Type selectors (e.g., div, p) have the lowest specificity.
  • Class selectors (e.g., .class-name) are more specific.
  • ID selectors (e.g., #id-name) are even more specific.
  • Inline styles (e.g., style="..." in the HTML) have the highest specificity, except for !important.
  • !important: Overrides everything, but should be used sparingly as it can make debugging difficult.

Example:

p { color: blue; }         /* Type selector */
.highlight { color: red; } /* Class selector */
#special { color: green; } /* ID selector */
p#special.highlight { color: yellow; } /* More specific */

2. Cascade and Order of Rules

What it is: When two rules have the same specificity, the last rule written in the CSS file will be applied (this is part of the Cascading in CSS).

Rule: Later rules override earlier ones if they have the same specificity.

Example:

p { color: blue; } 
p { color: green; } /* This will be applied since it's later */

3. Inheritance

What it is: Some CSS properties are inherited by child elements, while others are not.

  • Inherited properties: color, font-family, line-height, visibility, etc.
  • Non-inherited properties: margin, padding, border, background, width, etc.

Overriding inheritance: You can use the inherit keyword to explicitly make an element inherit a property.

    color: black; /* All text inside  inherits this color */
