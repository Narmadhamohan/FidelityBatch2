Networking Topics

TCP Model / IP Model

  • The TCP/IP Model is a foundational network model that governs data transmission over the Internet.
  • It consists of four layers: Application, Transport, Internet, and Link.
  • The model facilitates communication between diverse systems and networks.

ARPANet Architecture

  • ARPANET was the first operational packet-switching network and a precursor to the modern Internet.
  • It introduced key concepts such as distributed networking and the use of packet-switching technology.
  • ARPANET's architecture laid the groundwork for future network developments.

IP Addresses, Classes, and CIDR Notation

  • An IP address is a unique identifier assigned to each device on a network, enabling communication.
  • IP addresses are classified into several classes based on their network size and purpose:
    • Class A: Supports 16 million hosts; ranges from 1.0.0.0 to 126.255.255.255.
    • Class B: Supports 65,536 hosts; ranges from 128.0.0.0 to 191.255.255.255.
    • Class C: Supports 256 hosts; ranges from 192.0.0.0 to 223.255.255.255.
    • Class D: Used for multicast groups; ranges from 224.0.0.0 to 239.255.255.255.
    • Class E: Reserved for future use; ranges from 240.0.0.0 to 255.255.255.255.
  • CIDR (Classless Inter-Domain Routing) notation is a method for allocating IP addresses and routing IP packets.
  • CIDR notation uses a suffix (e.g., /24) to indicate the number of bits used for the network prefix:
    • For example, 192.168.1.0/24 means that the first 24 bits are the network part, allowing for 256 addresses in the range 192.168.1.0 to 192.168.1.255.

IPv4 and IPv6

  • IPv4 (Internet Protocol version 4) uses a 32-bit address space, allowing for approximately 4.3 billion unique addresses.
  • IPv4 addresses are expressed in decimal format, divided into four octets (e.g., 192.168.1.1).
  • IPv6 (Internet Protocol version 6) uses a 128-bit address space, allowing for a vastly larger number of unique addresses (approximately 340 undecillion addresses).
  • IPv6 addresses are expressed in hexadecimal format, separated by colons (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334).

Host Addressing, Multicast, and Broadcast

  • Host Addressing: Refers to the specific address assigned to a device on a network, which allows it to communicate with other devices.
  • Broadcast: A transmission method where data is sent to all devices on a network. In IPv4, the broadcast address for a subnet is typically the last address in the range.
  • Multicast: Allows a single packet to be sent to multiple devices, but only to those that are subscribed to the multicast address.

Subnet Calculation Example

  • Consider the IP address 192.168.1.0/24:
    • Subnet Mask: 255.255.255.0 (indicating that the first 24 bits are the network part).
    • Number of Hosts: 2^(32-24) - 2 = 254 (2 addresses are reserved for network and broadcast).
    • Valid Host Range: 192.168.1.1 to 192.168.1.254
    • Broadcast Address: 192.168.1.255

Protocols (IP, HTTP, TCP, FTP, ARP)

  • IP (Internet Protocol): Responsible for addressing and routing packets across networks.
  • HTTP (Hypertext Transfer Protocol): Governs communication between web browsers and servers.
  • Common Ports for HTTP:
    • Port 80 - HTTP
    • Port 443 - HTTPS (HTTP Secure)
  • TCP (Transmission Control Protocol): Ensures reliable data transfer and error recovery.
  • Common Ports for TCP:
    • Port 20 - FTP Data
    • Port 21 - FTP Control
    • Port 22 - SSH
    • Port 25 - SMTP
    • Port 53 - DNS
    • Port 110 - POP3
    • Port 143 - IMAP
    • Port 3306 - MySQL
    • Port 5432 - PostgreSQL
  • FTP (File Transfer Protocol): Used for transferring files between a client and server.
  • Common Ports for FTP:
    • Port 21 - FTP Control
    • Port 20 - FTP Data
  • ARP (Address Resolution Protocol): Resolves IP addresses to MAC (Media Access Control) addresses.

Importance of Ports

  • Ports are endpoints for communication, allowing multiple applications to use network resources simultaneously.
  • Common ports include:
    • Port 20 - FTP Data
    • Port 21 - FTP Control
    • Port 22 - SSH
    • Port 25 - SMTP
    • Port 53 - DNS
    • Port 80 - HTTP
    • Port 110 - POP3
    • Port 143 - IMAP
    • Port 443 - HTTPS
    • Port 3306 - MySQL
    • Port 5432 - PostgreSQL

TCP & UDP Ports Difference

  • TCP (Transmission Control Protocol) is connection-oriented, ensuring reliable and ordered delivery of data.
  • UDP (User Datagram Protocol) is connectionless, offering faster data transmission without guaranteeing delivery.
  • TCP is used for applications where reliability is crucial, while UDP is suitable for real-time applications like video streaming.
  • Common UDP Ports:
    • Port 53 - DNS
    • Port 67 - DHCP Server
    • Port 68 - DHCP Client
    • Port 69 - TFTP
    • Port 123 - NTP
    • Port 161 - SNMP
    • Port 162 - SNMP Trap
